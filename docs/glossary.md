# Glossary

- AI/ML: Artificial Intelligence / Machine Learning.
- AP: Access Point.
- Root AP: AP hosting or coordinating controller-side behavior in the local mesh context.
- Extender AP: Additional AP participating in the mesh.
- Multi-AP: Multi-access-point architecture associated with EasyMesh-style control.
- EasyMesh: Wi-Fi Alliance Multi-AP certification/program family; in this project treated through prplMesh/EasyMesh-like abstractions unless officially verified.
- IEEE 1905.1: Protocol used under Multi-AP/EasyMesh-style topology/control, not assumed to be arbitrary ML data transport.
- USP: User Services Platform, Broadband Forum TR-369.
- TR-181: Broadband Forum Device Data Model used with CWMP and USP Agents.
- USP Controller: Controller-side management component in USP architecture.
- USP Agent: Device-side component exposing service elements/data model.
- LXC: Linux Containers userspace/runtime family.
- LCM: Lifecycle Management; in this project used for containerized service deployment/management.
- prplMesh: prpl Foundation open-source EasyMesh implementation.
- prplOS: Open-source CPE/router platform from prpl Foundation ecosystem.
- Metrics store: Cloud-side receiver/index for ML data and service metrics.
- Anomaly resolver agent: Containerized agent that detects or responds to network anomalies; early stage may be rule-based only.
